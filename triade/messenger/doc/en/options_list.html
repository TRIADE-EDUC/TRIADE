_ACP_ALLOW_MEMORY_AUTH - new feature in version 2.0.5

Option not displayed on ACP.
If enabled, allows memory integrated authentication to ACP in web-browser cookies.
If option _ACP_PROTECT_BY_HTACCESS is not activated.

_ACP_PROTECT_BY_HTACCESS - new feature in version 2.0.5

Option not displayed on ACP.
If not enabled, allows (integrated authentication and) managing multiple access to ACP.
Each administrator can have different access rights.

_ADMIN_EMAIL

Set administrator email address.
Visible on client screen 'options': tab Server.
See also option _ADMIN_PHONE.

_ADMIN_PHONE

Set administrator phone number.
Visible on client screen 'options': tab Server.
See also option _ADMIN_EMAIL.

_ALLOW_AUTO_ADD_NEW_USER_ON_SERVER

Allow (or not) new users to be automatically added.
If option is not enabled, it's administrator to add users one by one.
See also options: _NEED_QUICK_REGISTER_TO_AUTO_ADD_NEW_USER - _PENDING_USER_ON_COMPUTER_CHANGE.
Recommended value: activated for a classic usage.

_ALLOW_CHANGE_AVATAR

Allow (or not) users to change their avatar (picture) (update profile).
Requires activation of option _ALLOW_MANAGE_PROFILE.

_ALLOW_CHANGE_CONTACT_NICKNAME

_ALLOW_CHANGE_EMAIL_PHONE

Allow (or not) user to update his phone number and email address (update profile).
Requires activation of option _ALLOW_MANAGE_PROFILE.

_ALLOW_CHANGE_FUNCTION_NAME

Allow (or not) user to update hirs function/name (update profile).
Requires activation of option _ALLOW_MANAGE_PROFILE.

_ALLOW_CLOSE_IM - new feature in version 2.0.4

Option not displayed on ACP.
Allow (or not) to close (quit) IntraMessenger client.
Recommended (default) value: activated.

_ALLOW_HIDDEN_TO_CONTACTS - option was named _ALLOW_INVISIBLE until version 2.0.4

_ALLOW_HISTORY_MESSAGES - option was named _ALLOW_USER_TO_HISTORY_MESSAGES until version 2.0.4

_ALLOW_HISTORY_MESSAGES_EXPORT - new feature in version 2.0.5

_AUTO_ADD_CONTACT_USER_ID - new feature in version 2.0.4

Option not displayed on ACP.
User's identifiers list for add to contacts at registration.
Typical usage: if IntraMessenger is used for helpdesk: add supports users in new users contacts list,
while blocking them to manage their contact list: see option _ALLOW_MANAGE_CONTACT_LIST.
Example: "1;15;20" (one to several values separated by semicolon).
Recommended value: unactivated (empty) for a classic usage.

_AWAY_REASONS_LIST

_BACKUP_FILES - new major feature in version 2.0.5

Allow (or not) Users files backup.

_BACKUP_FILES_ALLOW_MULTI_FOLDERS - new feature in version 2.0.5

Allow (or not) multi-folders backup. If not allowed, can backup only one folder.
Option _BACKUP_FILES_THIS_LOCAL_FOLDER_ONLY should not be activated.
See also option _BACKUP_FILES_ALLOW_SUB_FOLDERS.
Requires activation of option _BACKUP_FILES.
Recommended (default) value: activated.

_BACKUP_FILES_ALLOW_SUB_FOLDERS - new feature in version 2.0.5

Allow (or not) recursive backup (include sub-folders).
See also option _BACKUP_FILES_ALLOW_MULTI_FOLDERS.
Requires activation of option _BACKUP_FILES.
Recommended (default) value: activated.

_BACKUP_FILES_FOLDER - new feature in version 2.0.5

Relative path (from /admin/ to backup storage folder, if FTP access refer to IM web server.
With this option, admin can delete files from ACP.
If your FTP access refer to the folder /private_backup/files/ this option may to be ../private_backup/files/
If your FTP access refer to a folder /files/ upstair webiste root (/public_html/ or /www/) this option may to be ../../../files/
If (and only if) empty, fill option _BACKUP_FILES_FTP_PASSWORD.
Requires activation of option _BACKUP_FILES.
See also option _BACKUP_FILES_FTP_LOGIN.
Recommended value: path outside website root folder

_BACKUP_FILES_FORCE_EVERY_DAY_AT - new feature in version 2.0.5

Option not displayed on ACP.
Time to automatically backup files (24 hours format, example: 23:59).
The backup is forced Force backup every day at HH:MM (24 hours format, example: 23:59).
Requires activation of option _BACKUP_FILES.
See also option _BACKUP_FILES_THIS_LOCAL_FOLDER_ONLY.

_BACKUP_FILES_FTP_ADDRESS - new feature in version 2.0.5

Your FTP server address (example: ftp.yourserver.com) for files backup.
Requires activation of option _BACKUP_FILES.
See also option _BACKUP_FILES_FTP_PORT_NUMBER.

_BACKUP_FILES_FTP_LOGIN - new feature in version 2.0.5

FTP login (username) for files backup.
Check that this login cannot access to your root website, but only one folder (example: /intramessenger/private_backup/files/).
Better to share a FTP folder outside website (not in a /www/ or /public_html/ subfolder).
Requires activation of option _BACKUP_FILES.
See also options _BACKUP_FILES_FTP_PASSWORD_CRYPT and _BACKUP_FILES_FTP_ADDRESS.

_BACKUP_FILES_FTP_PASSWORD - new feature in version 2.0.5

Only if FTP on another server (so option _BACKUP_FILES_FOLDER empty): FTP password (clear).
With this option, admin can delete files from ACP.
Requires activation of option _BACKUP_FILES.
See also options _BACKUP_FILES_FTP_LOGIN and _BACKUP_FILES_FTP_PASSWORD_CRYPT.

_BACKUP_FILES_FTP_PASSWORD_CRYPT - new feature in version 2.0.5

FTP password crypted by IM_skin (skin kit).
Requires activation of option _BACKUP_FILES.
See also options _BACKUP_FILES_FTP_LOGIN and _BACKUP_FILES_FTP_PASSWORD.

_BACKUP_FILES_FTP_PORT_NUMBER - new feature in version 2.0.5

Requires activation of option _BACKUP_FILES.
See also option _BACKUP_FILES_FTP_ADDRESS.
Recommended (default) value: 21.

_BACKUP_FILES_MAX_ARCHIVE_SIZE - new feature in version 2.0.5

Max archive/backup file size (MB).
Requires activation of option _BACKUP_FILES.

_BACKUP_FILES_MAX_NB_ARCHIVES_USER - new feature in version 2.0.5

Max backup/archives (dump) number stored per user (1 to 9).
Requires activation of option _BACKUP_FILES.
Recommended (default) value: 3.

_BACKUP_FILES_MAX_SPACE_SIZE_TOTAL - new feature in version 2.0.5

Max backup archives storage space (MB).
Requires activation of option _BACKUP_FILES.

_BACKUP_FILES_MAX_SPACE_SIZE_USER - new feature in version 2.0.5

Max backup archives storage space (MB) per user.
Requires activation of option _BACKUP_FILES.

_BACKUP_FILES_THIS_LOCAL_FOLDER_ONLY - new feature in version 2.0.5

Option not displayed on ACP.
Backup this local folder only (empty: user choose).
Option _BACKUP_FILES_ALLOW_MULTI_FOLDERS should not be activated.
See also option _BACKUP_FILES_ALLOW_SUB_FOLDERS and _BACKUP_FILES_FORCE_EVERY_DAY_AT.

_LOCK_AFTER_NO_ACTIVITY_DURATION

Set after how many days of inactivity an account is automatically locked (0: unlimited).
These ghost accounts needlessly monopolize server resources.
Do not confuse last activity date and last access date.
Send/receive a message, add a contact updates the last activity date.
See also options: _LOCK_AFTER_NO_CONTACT_DURATION - _OUTOFDATE_AFTER_NOT_USE_DURATION.
Recommended value: 90.

_LOCK_AFTER_NO_CONTACT_DURATION - new feature in version 2.0.4

_SHARE_FILES - major feature in version 2.0.4

See also option _SHARE_FILES_EXCHANGE for exchange private files (between users).

_SHARE_FILES_ALLOW_UPPERCASE

Requires activation of option _SHARE_FILES.
Recommended value: not activated, mainly on Windows FTP server!

_SHARE_FILES_FOLDER

With this option, admin can download files from ACP.
If your FTP access refer to the folder /share/files/ this option may to be ../share/files/
If your FTP access refer to a folder /files/ upstair webiste root (/public_html/ or /www/) this option may to be ../../../files/
If (and only if) empty, fill option _SHARE_FILES_FTP_PASSWORD.
Requires activation of option _SHARE_FILES.

_SLOW_NOTIFY - option was named _FULL_CHECK until version 2.0.4

Force (or not) to check all the time for pending contacts.
If activated, check only every 3 minutes, to limit requests to the server (especially if number of users is important).
See also option _CHECK_NEW_MSG_EVERY.

_SPECIAL_MODE_GROUP_COMMUNITY

Active (or not) the special mode: everybody can see everybody (only) of they are in the SAME GROUP(s).
Group management is centralized by the administrator: see also option _GROUP_USER_CAN_JOIN.
To note: a user isnt't in one (or more) group, he cannot see anybody (and nobody see him).
See also option _GROUP_ID_DEFAULT_FOR_NEW_USER.
Options: _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY - _GROUP_FOR_SBX_AND_ADMIN_MSG should not be activated.
Recommended value: unactivated for a classic usage.

_SPECIAL_MODE_OPEN_COMMUNITY

Active (or not) the special mode: everybody can see everybody, without have to adding to their contact list.
Options: _SPECIAL_MODE_GROUP_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY should not be activated.
Recommended value: unactivated for a classic usage.

_SPECIAL_MODE_OPEN_GROUP_COMMUNITY

Active (or not) the special mode: everybody can see everybody (without adding to their contact list).
Order by groups, managed by administrator: see also option _GROUP_USER_CAN_JOIN.
See also option _GROUP_ID_DEFAULT_FOR_NEW_USER.
Options: _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_GROUP_COMMUNITY - _GROUP_FOR_SBX_AND_ADMIN_MSG should not be activated.
Recommended value: unactivated for a classic usage.

_STATISTICS

Option not displayed on ACP.
Archive (or not) statistics. Consultation in ACP.
Recommended value: activated.

_STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL

Option not displayed on ACP.
Set new IntraMessenger server address (url): users will switch immediately on next startup.
Attention, this is not the address of the server that is rename, it's a new address (in addition) !
Changing address example on the same web server (so IntraMessenger installed twice):
    http://your-server/intramessenger/
    http://your-server/im/
If activated, hide option: _STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL_AFTER_LOGIN!

_STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL_AFTER_LOGIN - new feature in version 2.0.5

Option not displayed on ACP.
Similar to option _STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL must not be activated.
Set new IntraMessenger server address: users will switch on next startup after authentication.
The address change is therefore known only to authenticated users.

_TIME_ZONES

Display (or not) time zones.
Option added to disable this feature (added in version 1.3.5).
Recommended value: enabled only for use on internet.

_UNREAD_MESSAGE_VALIDITY

Set unread messages validity (days) (0: unlimited) before expiration (deletion).
Recommended value: 30.

_USER_HIEARCHIC_MANAGEMENT_BY_ADMIN

Active (or not) the special mode: hierarchical users management, display level column in ACP.
User cannot request to be added in contact list of superior level user.
Levels defined in server language file (/common/lang/) in level area.
Default level are: Administrator - Director - Manager - Employee - Guest (define number with $c_nb_level).
A guest cannot request to a Employee, while the opposite is possible.
Requires activation of option _ALLOW_MANAGE_CONTACT_LIST.
Options: _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY should not be activated.
Recommended value: unactivated for a classic usage.

_USER_NEED_PASSWORD

Force (or not) use password for each user.
Can disable if one of this options _FORCE_USERNAME_TO_PC_SESSION_NAME or _PENDING_USER_ON_COMPUTER_CHANGE is activated.
See also options: _MINIMUM_PASSWORD_LENGTH - _MAX_PASSWORD_ERRORS_BEFORE_LOCK_USER - _EXTERNAL_AUTHENTICATION - _PASSWORD_FOR_PRIVATE_SERVER.
Recommended value: activated.

_WAIT_STARTUP_IF_SERVER_UNAVAILABLE

Active (or not) client still wait (without warn) if server unavailable on startup (instead of stop).

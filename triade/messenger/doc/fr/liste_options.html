_ACP_ALLOW_MEMORY_AUTH - nouveauté de la version 2.0.5

Option non affichée dans l'ACP.
Si activée, permet la mémorisation de l'authentification intégrée à l'ACP dans les cookies du navigateur web.
Si l'option _ACP_PROTECT_BY_HTACCESS n'est pas activée.

_ACP_PROTECT_BY_HTACCESS - nouveauté de la version 2.0.5

Option non affichée dans l'ACP.
Si non activée, permet (l'authentification intégrée et) la gestion d'accès multiples à l'ACP.
Chaque administrateur pouvant avoir des droits d'accès différents.
Les administrateurs des versions antérieures à 2.0.5 pourront activer cette option pour conserver la méthode d'authentification habituelle (via htaccess).

_ADMIN_EMAIL

Définir l'adresse email de l'administrateur.
Visible sur les postes clients dans l'onglet Serveur.
Voir aussi l'option _ADMIN_PHONE.

_ADMIN_PHONE

Définir le numéro de téléphone de l'administrateur.
Visible sur les postes clients dans l'onglet Serveur.
Voir aussi l'option _ADMIN_EMAIL.

_ALLOW_AUTO_ADD_NEW_USER_ON_SERVER

Autoriser (ou non) les nouveaux utilisateurs à être automatiquement ajoutés.
Si l'option n'est pas activée, c'est l'administrateur qui ajoute (créé) les utilisateurs un par un.
Voir aussi les options : _NEED_QUICK_REGISTER_TO_AUTO_ADD_NEW_USER - _PENDING_USER_ON_COMPUTER_CHANGE.
Valeur conseillée : activée pour une utilisation classique.

_ALLOW_CHANGE_AVATAR

Autoriser (ou non) l'utilisateur à changer son avatar (photo) en modifiant son profil.
Nécessite l'activation de l'option _ALLOW_MANAGE_PROFILE.

_ALLOW_CHANGE_CONTACT_NICKNAME

Autoriser (ou non) l'utilisateur à renommer ses contacts.
Les options : _ALLOW_MANAGE_CONTACT_LIST - _ALLOW_MANAGE_PROFILE doivent être activées.

_ALLOW_CHANGE_EMAIL_PHONE

Autoriser (ou non) l'utilisateur à changer son numéro de téléphone et son adresse email (en modifiant son profil).
Nécessite l'activation de l'option _ALLOW_MANAGE_PROFILE.

_ALLOW_CHANGE_FUNCTION_NAME

Autoriser (ou non) l'utilisateur à changer sa fonction (en modifiant son profil).
Nécessite l'activation de l'option _ALLOW_MANAGE_PROFILE.

_ALLOW_CLOSE_IM - nouveauté de la version 2.0.4

Option non affichée dans l'ACP.
Autoriser (ou non) à fermer le client IntraMessenger
Valeur (par défaut) conseillée : activée.

_ALLOW_COL_FUNCTION_NAME - option (inversée) anciennement nommée _HIDE_COL_FUNCTION_NAME jusqu'à la version 2.0.4

Option non affichée dans l'ACP.
Autoriser (ou non) les utilisateurs à afficher la colonne nom/fonction.

_ALLOW_CONFERENCE

Autoriser (ou non) les utilisateurs à se créer des conférences à plusieurs.

_ALLOW_CONTACT_RATING - option anciennement nommée _ALLOW_USER_RATING jusqu'à la version 2.0.4

Autoriser (ou non) l'utilisateur à noter ses contacts.
Si l'option est définie à PUBLIC, chacun peut consulter sa moyenne.
Nécessite l'activation de l'option _ALLOW_MANAGE_CONTACT_LIST.

_ALLOW_EMAIL_NOTIFIER

Autoriser (ou non) l'usage du notifieur d'email intégré.
Nécessite l'activation de l'option _ALLOW_MANAGE_OPTIONS.
Voir aussi l'option _INCOMING_EMAIL_SERVER_ADDRESS.

_ALLOW_HIDDEN_STATUS - nouveauté de la version 2.0.5

Autoriser (ou non) l'usage de l'état caché (indiqué "non connecté" pour tous).
Voir aussi l'option _ALLOW_HIDDEN_TO_CONTACTS.

_ALLOW_HIDDEN_TO_CONTACTS - option anciennement nommée _ALLOW_INVISIBLE jusqu'à la version 2.0.4

Autoriser (ou non) l'utilisateur à être invisible (caché si en ligne) pour certains de ses contacts (voir VIP).
Voir aussi les options _ALLOW_MANAGE_CONTACT_LIST et _ALLOW_HIDDEN_STATUS.

_ALLOW_HISTORY_MESSAGES - option anciennement nommée _ALLOW_USER_TO_HISTORY_MESSAGES jusqu'à la version 2.0.4

Autoriser (ou non) l'utilisateur à archiver (sur son poste) les messages échangés.
Si autorisé, l'utilisateur défini dans ses options combien de jours il souhaite les conserver (0 : pas d'archivage).
Voir aussi l'option _ALLOW_HISTORY_MESSAGES_EXPORT.

_ALLOW_HISTORY_MESSAGES_EXPORT - nouveauté de la version 2.0.5

Autoriser (ou non) l'utilisateur à exporter (au format Excel) l'historique des messages archivés.
L'option _ALLOW_HISTORY_MESSAGES doit être activée.

_ALLOW_MANAGE_CONTACT_LIST - option (inversée) anciennement nommée _LOCK_USER_CONTACT_LIST jusqu'à la version 2.0.4

Empêcher (ou non) les utilisateurs d'accèder à leur liste de contacts ainsi qu'à leur alarme.
Si activée, seul l'administrateur peut leur ajouter/supprimer des contacts.
Option de verrouillage utilisée par exemple en milieu scolaire.
Voir aussi les options : _SPECIAL_MODE_GROUP_COMMUNITY - _ALLOW_CHANGE_CONTACT_NICKNAME - _ALLOW_CONTACT_RATING - _PUBLIC_USERS_LIST.

_ALLOW_MANAGE_OPTIONS - option (inversée) anciennement nommée _LOCK_USER_OPTIONS jusqu'à la version 2.0.4

Empêcher (ou non) les utilisateurs d'accèder à leurs options ainsi qu'à leur alarme.
Option de verrouillage utilisée par exemple en milieu scolaire.
Voir aussi les options : _ALLOW_EMAIL_NOTIFIER - _ALLOW_SKIN - _ALLOW_SOUND_USAGE - _PUBLIC_OPTIONS_LIST.

_ALLOW_MANAGE_PROFILE - option (inversée) anciennement nommée _LOCK_USER_PROFILE jusqu'à la version 2.0.4

Empêcher (ou non) les utilisateurs d'accèder à leur profil.
Option de verrouillage utilisée par exemple en milieu scolaire.
Voir aussi les options : _ALLOW_CHANGE_AVATAR - _ALLOW_CHANGE_EMAIL_PHONE - _ALLOW_CHANGE_FUNCTION_NAME - _INVITE_FILL_PROFILE_ON_FIRST_LOGIN.

_ALLOW_POST_IT - nouveauté de la version 2.0.4

Autoriser (ou non) l'utilisateur à utiliser des post-it.

_ALLOW_REDUCE_MAIN_SCREEN - nouveauté de la version 2.0.4

Option non affichée dans l'ACP.
Autoriser (ou non) à réduire la fenêtre principale.
Valeur (par défaut) conseillée : activée.

_ALLOW_REDUCE_MESSAGE_SCREEN - nouveauté de la version 2.0.4

Option non affichée dans l'ACP.
Autoriser (ou non) à réduire la fenêtre de discussion.
Valeur (par défaut) conseillée : activée.

_ALLOW_SEND_TO_OFFLINE_USER

Autoriser (ou non) l'utilisateur à envoyer des messages à un contact non connecté.
(ne permet pas l'envoi à un groupe d'utilisateurs non connectés).

_ALLOW_SKIN - nouveauté de la version 2.0.4

Autoriser (ou non) à appliquer un skin sur le client IntraMessenger
Nécessite l'activation de l'option _ALLOW_MANAGE_OPTIONS.
Valeur (par défaut) conseillée : activée.

_ALLOW_SMILEYS

Autoriser (ou non) l'utilisateur à envoyer des émoticones (smileys, affichés en images) pendant une discussion.

_ALLOW_SOUND_USAGE - nouveauté de la version 2.0.4

Autoriser (ou non) à utiliser les sons (notifications).
Nécessite l'activation de l'option _ALLOW_MANAGE_OPTIONS.
Valeur (par défaut) conseillée : activée.

_ALLOW_UPPERCASE_SPACE_USERNAME

Option non affichée dans l'ACP.
Autoriser (ou non) l'utilisateur à avoir dans son identifiant (pseudo) des majuscules et/ou caractères espaces.
Voir aussi l'option _FORCE_USERNAME_TO_PC_SESSION_NAME.
Valeur conseillée : activée.

_ALLOW_USE_PROXY

Autoriser (ou non) l'utilisateur à paramètrer l'accès au serveur IntraMessenger via un serveur proxy.
Voir aussi les options : _PROXY_ADDRESS - _PROXY_PORT_NUMBER.

_AUTO_ADD_CONTACT_USER_ID - nouveauté de la version 2.0.4

Option non affichée dans l'ACP.
Liste des identifiants des utilisateurs à ajouter aux contacts dès l'inscription.
Usage typique : si IntraMessenger vous sert à effectuer du support client (helpdesk) : ajouter comme contacts les personnes du supports à nouveaux utilisateurs,
tout en leur bloquant la gestion de leur liste de contact : voir l'option : _ALLOW_MANAGE_CONTACT_LIST.
Exemple de valeur : "1;15;20" (une à plusieurs valeurs séparées par le caractère point-virgule).
Valeur conseillée : désactivée (vide) pour une utilisation classique.

_AWAY_REASONS_LIST

Permet d'ajouter une série de motifs pour l'état absent.
Jusqu'à 10 motifs séparés par des point-virgules.
La liste des motifs est la même quelque soit la langue configurée par l'utilisateur.
Options similaires : _ONLINE_REASONS_LIST - _BUSY_REASONS_LIST - _DONOTDISTURB_REASONS_LIST.
Voir aussi l'option _FORCE_STATUS_LIST_FROM_SERVER.

_BACKUP_FILES - nouveauté majeure de la version 2.0.5

Autoriser (ou non) la sauvegarde des fichiers utilisateurs.

_BACKUP_FILES_ALLOW_MULTI_FOLDERS - nouveauté de la version 2.0.5

Autoriser (ou non) la sauvegarde de plusieurs répertoires. Si non autorisé, permet de sauvegarder un seul répertoire uniquement.
L'option _BACKUP_FILES_THIS_LOCAL_FOLDER_ONLY ne doit pas être activée.
Voir aussi l'option : _BACKUP_FILES_ALLOW_SUB_FOLDERS.
Nécessite l'activation de l'option _BACKUP_FILES.
Valeur (par défaut) conseillée : activée.

_BACKUP_FILES_ALLOW_SUB_FOLDERS - nouveauté de la version 2.0.5

Autoriser (ou non) la sauvegarde (récursive) des sous-dossiers.
Voir aussi l'option : _BACKUP_FILES_ALLOW_MULTI_FOLDERS.
Nécessite l'activation de l'option _BACKUP_FILES.
Valeur (par défaut) conseillée : activée.

_BACKUP_FILES_FOLDER - nouveauté de la version 2.0.5

Chemin relatif (depuis /admin/) d'accès aux sauvegardes, si les accès FTP correspondent à un répertoire sur le serveur web.
Avec cette option renseignée, les fichiers sont supprimables depuis l'interface d'administration (ACP).
Si l'accès FTP correspondent au dossier /private_backup/files/ cette option devrait être ../private_backup/files/
Si l'accès FTP correspondent à dossier /files/ au dessus de la racine site web (/public_html/ ou /www/) cette option devrait être ../../../files/
Si (et uniquement si) vide, renseigner l'option _BACKUP_FILES_FTP_PASSWORD.
Nécessite l'activation de l'option _BACKUP_FILES.
Voir aussi l'option _BACKUP_FILES_FTP_LOGIN.
Valeur conseillée : chemin au dessus de la racine du site web

_BACKUP_FILES_FORCE_EVERY_DAY_AT - nouveauté de la version 2.0.5

Option non affichée dans l'ACP.
Heure de sauvegarde automatique (format 24 heures, exemple: 23:59).
La sauvegarde est forcée tous les jours à cette heure là.
Nécessite l'activation de l'option _BACKUP_FILES.
Voir aussi l'option _BACKUP_FILES_THIS_LOCAL_FOLDER_ONLY.

_BACKUP_FILES_FTP_ADDRESS - nouveauté de la version 2.0.5

Adresse de votre serveur FTP (exemple : ftp.votreserveur.com) de sauvegarde des fichiers utilisateurs.
Nécessite l'activation de l'option _BACKUP_FILES.
Voir aussi l'option _BACKUP_FILES_FTP_PORT_NUMBER.

_BACKUP_FILES_FTP_LOGIN - nouveauté de la version 2.0.5

Login d'accès FTP de sauvegarde des fichiers utilisateurs.
Vérifier que ce login n'a pas accès à la racine de votre site web, mais uniquement à un seul dossier (exemple : /intramessenger/private_backup/files/).
Il est préférable de partager un dossier FTP en dehors du site web (pas un sous-dossier de /www/ ou /public_html/).
Nécessite l'activation de l'option _BACKUP_FILES.
Voir aussi les options _BACKUP_FILES_FTP_PASSWORD_CRYPT et _BACKUP_FILES_FTP_ADDRESS.

_BACKUP_FILES_FTP_PASSWORD - nouveauté de la version 2.0.5

Uniquement si FTP sur autre serveur (donc option _BACKUP_FILES_FOLDER vide) : mot de passe (en clair) de l'accès FTP.
Avec cette option renseignée, les fichiers sont supprimables depuis l'interface d'administration (ACP).
Nécessite l'activation de l'option _BACKUP_FILES.
Voir aussi les options _BACKUP_FILES_FTP_LOGIN et _BACKUP_FILES_FTP_PASSWORD_CRYPT.

_BACKUP_FILES_FTP_PASSWORD_CRYPT - nouveauté de la version 2.0.5

Mot de passe de l'accès FTP chiffré par IM_skin (kit skin).
Nécessite l'activation de l'option _BACKUP_FILES.
Voir aussi les options _BACKUP_FILES_FTP_LOGIN et _BACKUP_FILES_FTP_PASSWORD.

_BACKUP_FILES_FTP_PORT_NUMBER - nouveauté de la version 2.0.5

Nécessite l'activation de l'option _BACKUP_FILES.
Voir aussi l'option _BACKUP_FILES_FTP_ADDRESS.
Valeur (par défaut) conseillée : 21.

_BACKUP_FILES_MAX_ARCHIVE_SIZE - nouveauté de la version 2.0.5

Taille maxi par sauvegarde/archive (en Mo).
Nécessite l'activation de l'option _BACKUP_FILES.

_BACKUP_FILES_MAX_NB_ARCHIVES_USER - nouveauté de la version 2.0.5

Nombre maxi de sauvegardes/archives (dump) stockées par utilisateur (de 1 à 9).
Nécessite l'activation de l'option _BACKUP_FILES.
Valeur (par défaut) conseillée : 3.

_BACKUP_FILES_MAX_SPACE_SIZE_TOTAL - nouveauté de la version 2.0.5

Taille maxi de l'espace d'hébergement des sauvegardes (en Mo).
Nécessite l'activation de l'option _BACKUP_FILES.

_BACKUP_FILES_MAX_SPACE_SIZE_USER - nouveauté de la version 2.0.5

Taille maxi de l'espace d'hébergement des sauvegardes (en Mo) par utilisateur.
Nécessite l'activation de l'option _BACKUP_FILES.

_BACKUP_FILES_THIS_LOCAL_FOLDER_ONLY - nouveauté de la version 2.0.5

Option non affichée dans l'ACP.
Sauvegarder uniquement ce répertoire local (vide : choix par l'utilisateur).
L'option _BACKUP_FILES_ALLOW_MULTI_FOLDERS ne doit pas être activée.
Voir aussi les options : _BACKUP_FILES_ALLOW_SUB_FOLDERS et _BACKUP_FILES_FORCE_EVERY_DAY_AT.
Nécessite l'activation de l'option _BACKUP_FILES.
Valeur (par défaut) conseillée : vide.

_BOOKMARKS

Activer (ou non) le partage de marques pages (favoris).

_BOOKMARKS_NEED_APPROVAL

Rendre (ou non) les marques pages partagés modérés (approuvés avant publication).
Nécessite l'activation de l'option _BOOKMARKS.

_BOOKMARKS_PUBLIC

Rendre (ou non) les marques pages partagés consultables par tout le monde (abonnement flux RSS disponible).
Nécessite l'activation de l'option _BOOKMARKS.

_BOOKMARKS_VOTE

Autoriser (ou non) la possibilité de voter (créer le buzz) des marques pages partagés.
Nécessite l'activation de l'option _BOOKMARKS.

_BUSY_REASONS_LIST

Permet d'ajouter une série de motifs pour l'état occupé.
Jusqu'à 10 motifs séparés par des point-virgules.
La liste des motifs est la même quelque soit la langue configurée par l'utilisateur.
Options similaires : _ONLINE_REASONS_LIST - _AWAY_REASONS_LIST - _DONOTDISTURB_REASONS_LIST.
Voir aussi l'option _FORCE_STATUS_LIST_FROM_SERVER.

_CENSOR_MESSAGES

Activer (ou non) la censure des messages échangés via un dictionnaire de mots interdits contenus dans le fichier /common/config/censure.txt
L'option _CRYPT_MESSAGES ne doit pas être activée.

_CHECK_NEW_MSG_EVERY

Définir l'interval de vérification de l'arrivée d'un premier nouveau message (10 à 60 secondes).
Permet de limiter les accès/requêtes au serveur (surtout si le nombre d'utilisateurs est important).
Voir aussi l'option _SLOW_NOTIFY.
Valeur conseillée : 30.

_CHECK_VERSION_INTERNET

Activer (ou non) la vérification sur internet de la disponibilité d'une nouvelle version serveur : indication affichée dans le tableau de bord.
Depuis la version 2.0.3 cette vérification n'est effectuée qu'une fois par jour au maximum.
Valeur conseillée : activation vivement recommandée si le serveur a accès à internet (désactiver dans le cas contraire).

_CRYPT_MESSAGES

Activer (ou non) le chiffrement assymétrique des messages échangés entre les utilisateurs.
Les options : _CENSOR_MESSAGES - _HISTORY_MESSAGES_ON_ACP ne doivent pas être activées.

_DONOTDISTURB_REASONS_LIST

Permet d'ajouter une série de motifs pour l'état ne pas déranger.
Jusqu'à 10 motifs séparés par des point-virgules.
La liste des motifs est la même quelque soit la langue configurée par l'utilisateur.
Options similaires : _ONLINE_REASONS_LIST - _AWAY_REASONS_LIST - _BUSY_REASONS_LIST.
Voir aussi l'option _FORCE_STATUS_LIST_FROM_SERVER.

_ENTERPRISE_SERVER

Activer (ou non) le mode entreprise : remonté des versions de logiciels installés, possibilité d'envoyer des ordres (arrêt, redémarrage...) à distance.
Voir aussi l'option _GROUP_FOR_SBX_AND_ADMIN_MSG.

_EXTERN_URL_CHANGE_PASSWORD

Définir l'adresse (URL) pour changer son mot de passe (via l'authentification externe/tierce).
Uniquement si l'option _EXTERNAL_AUTHENTICATION est activée.
Nécessite l'activation de l'option _USER_NEED_PASSWORD.

_EXTERN_URL_FORGET_PASSWORD

Adresse (URL) pour récupérer son mot de passe oublié (via l'authentification externe/tierce).
Uniquement si l'option _EXTERNAL_AUTHENTICATION est activée.
Nécessite l'activation de l'option _USER_NEED_PASSWORD.

_EXTERN_URL_TO_REGISTER

Adresse (URL) pour s'inscrire (forum, CMS...) via l'authentification externe/tierce.
Uniquement si l'option _EXTERNAL_AUTHENTICATION est activée.

_EXTERNAL_AUTHENTICATION

Configurer (ou non) l'authentification externe/tierce via une base de données existantes. Choix dans une liste prédéfinie.
Nécessite l'activation des options : _USER_NEED_PASSWORD - _ALLOW_AUTO_ADD_NEW_USER_ON_SERVER.
Les options : _PENDING_NEW_AUTO_ADDED_USER - _NEED_QUICK_REGISTER_TO_AUTO_ADD_NEW_USER ne doivent pas être activées.
Voir aussi les options : _EXTERN_URL_CHANGE_PASSWORD - _EXTERN_URL_FORGET_PASSWORD - _EXTERN_URL_TO_REGISTER.

_FLAG_COUNTRY_FROM_IP - option anciennement nommée _DISPLAY_USER_FLAG_COUNTRY jusqu'à la version 2.0.4

Activer (ou non) l'affichage du drapeau du pays correspondant à l'adresse IP de l'utilisateur.
Valeur conseillée : activée uniquement pour un usage sur internet.

_FORCE_AWAY_ON_SCREENSAVER

Forcer (ou non) l'état absent lorsque l'économiseur d'écran est actif (ou la session verrouillée).
L'utilisateur ne peut plus désactiver cette option.

_FORCE_LAUNCH_ON_STARTUP - nouveauté de la version 2.0.4

Option non affichée dans l'ACP.
Forcer (ou non) le démarrage au lancement de Windows (voir ici).

_FORCE_OPTION_FILE_FROM_SERVER - nouveauté de la version 2.0.5.241

Option non affichée dans l'ACP.
Forcer (ou non) le fichier d'options depuis le serveur.
Mettre le fichier options.ini dans le dossier /distant/update/ (without username=... mais avec url=...).
Nécessite au moins le client 1.31D

_FORCE_STATUS_LIST_FROM_SERVER

Forcer (ou non) les status (disponible, absent, occupé...) à ceux contenus dans le fichier de la langue configurée (option _LANG) sur le serveur.
La liste des status est donc la même quelque soit la langue configurée par l'utilisateur.
Voir aussi les options : _ONLINE_REASONS_LIST - _AWAY_REASONS_LIST - _BUSY_REASONS_LIST - _DONOTDISTURB_REASONS_LIST.

_FORCE_UPDATE_BY_INTERNET

Forcer (ou non) les mises à jour des postes clients depuis le site internet officiel.
L'utilisateur ne peut plus désactiver la mise à jour.
Valeur conseillée : activée si les postes ont tous accès à internet.

_FORCE_UPDATE_BY_SERVER

Forcer (ou non) les mises à jour des postes clients depuis votre serveur IntraMessenger.
Voir cette documentation : /doc/fr/comment_mettre_a_jour_les_postes.html.
L'utilisateur ne peut plus désactiver la mise à jour.
Valeur conseillée : activée si les postes n'ont pas tous accès à internet.

_FORCE_USERNAME_TO_PC_SESSION_NAME

Forcer (ou non) l'identifiant (pseudo) au nom de session Windows.
Si tous les postes sont en domaine, en activant cette option l'accès sans mot de passe (option _USER_NEED_PASSWORD) devient possible.
Si non activée : choix du pseudo par l'utilisateur.
Voir aussi les options : _PENDING_USER_ON_COMPUTER_CHANGE - _MINIMUM_USERNAME_LENGTH - _ALLOW_UPPERCASE_SPACE_USERNAME.
Valeur conseillée : désactivée pour une utilisation classique.

_GROUP_FOR_SBX_AND_ADMIN_MSG

Activer (ou non) l'utilisation des groupes (gestion par l'administrateur) pour la ShoutBox (exemple) et grouper des envois de messages administrateur.
Les options _SPECIAL_MODE_GROUP_COMMUNITY et _SPECIAL_MODE_OPEN_GROUP_COMMUNITY ne doivent pas être activée.
Voir aussi les options : _GROUP_USER_CAN_JOIN - _GROUP_ID_DEFAULT_FOR_NEW_USER - _SHOUTBOX.

_GROUP_ID_DEFAULT_FOR_NEW_USER - nouveauté de la version 2.0.5.241

Option non affichée dans l'ACP.
Numéro du groupe dans lequel est automatiquement ajouté chaque nouvel utilisateur.
Une seule de ces 3 options : _SPECIAL_MODE_GROUP_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY - _GROUP_FOR_SBX_AND_ADMIN_MSG doit être activée.
Voir aussi les options : _GROUP_USER_CAN_JOIN - _GROUP_FOR_SBX_AND_ADMIN_MSG.

_GROUP_USER_CAN_JOIN

Autoriser (ou non) :
  - les utilisateurs à rejoindre des groupes publics.
  - les utilisateurs à demander de rejoindre des groupes officiels (validation par l'administrateur ou les utilisateurs ayant ce rôle activé).
Une seule de ces 3 options : _SPECIAL_MODE_GROUP_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY - _GROUP_FOR_SBX_AND_ADMIN_MSG doit être activée.
Voir aussi les options : _GROUP_ID_DEFAULT_FOR_NEW_USER - _GROUP_FOR_SBX_AND_ADMIN_MSG.

_HISTORY_MESSAGES_ON_ACP - option anciennement nommée _LOG_MESSAGES jusqu'à la version 2.0.4

Archiver (ou non) sur le serveur les messages échangés.
Consultation depuis l'interface d'administration (ACP).
L'option _CRYPT_MESSAGES ne doit pas être activée.
Voir aussi l'option _CENSOR_MESSAGES.

_IM_ADDRESS_BOOK_PASSWORD

Définir le mot de passe pour s'inscrire à l’annuaire des serveurs publics sur internet.
Etre dans l'annuaire permet aux utilisateurs d'accéder à votre serveur directement depuis la liste déroulante.
Ce mot de passe confirme votre choix de publier l'adresse de votre serveur IntraMessenger. En le modifiant vous indiquez ne plus vouloir être publié.
Si vous l'avez modifié par erreur et n'êtes plus enregistré sur l'annuaire, inscrivez vous à nouveau.

_INCOMING_EMAIL_SERVER_ADDRESS

Définir (forcer) l'adresse du serveur de courrier entrant (pour le notifieur intégré).
Nécessite l'activation de l'option _ALLOW_EMAIL_NOTIFIER.

_INVITE_FILL_PROFILE_ON_FIRST_LOGIN

Inviter (ou non) l'utilisateur à renseigner son profil à la première utilisation.
Nécessite l'activation de l'option _ALLOW_MANAGE_PROFILE.

_LANG

Choisir la langue par défaut de l'interface d'administration (ACP) et de certaines informations transmises aux utilisateurs.

_LOCK_AFTER_NO_ACTIVITY_DURATION

Définir après combien de jours d'inactivité un compte est automatiquement verrouillé (0 : illimité).
Ces comptes fantômes monopolisent inutilement des ressources serveur.
Ne pas confondre la date dernière activité avec la date de dernier accès.

Définir le quota hebdomadaire de messages par utilisateur (0 : illimité).
Nécessite MySQL version 5 minimum.
Nécessite l'activation de l'option _SHOUTBOX.

_SHOUTBOX_REFRESH_DELAY

Définir le délai de rafraichissement de la ShoutBox (10 à 180 secondes).
Permet de limiter les accès/requêtes au serveur (surtout si le nombre d'utilisateurs est important).
Nécessite l'activation de l'option _SHOUTBOX.
Valeur conseillée : 50.

_SHOUTBOX_REMOVE_MESSAGE_VOTES

Définir le nombre de votes négatifs activant la suppression automatique du message (0 : illimité).
Modération (auto-censure) par l'ensemble des participants.
Nécessite l'activation des options _SHOUTBOX - _SHOUTBOX_VOTE.

_SHOUTBOX_STORE_DAYS

Définir la durée (jours) de stockage des messages dans la ShoutBox (avant expiration).
Nécessite l'activation de l'option _SHOUTBOX.

_SHOUTBOX_STORE_MAX

Définir le nombre maximum de messages stockés dans la ShoutBox (ne conserver que les plus récents).
Nécessite l'activation de l'option _SHOUTBOX.

_SHOUTBOX_VOTE

Activer (ou non) la possibilité de noter (voter pour ou contre) les messages échangés dans la ShoutBox.
Nécessite l'activation de l'option _SHOUTBOX.

_SITE_TITLE

Définir le titre du site afin de remplacer le nom de la plateforme utilisée (ex: phpBB, WordPress...) en authentification externe/tierce.
Uniquement si l'option _EXTERNAL_AUTHENTICATION est activée.

_SITE_TITLE_TO_SHOW

Définir le titre du site à afficher en splash screen ainsi que dans l'onglet à propos (sauf si l'onglet est volontairement masqué en version personnalisée).
Si cette option ainsi que _SITE_URL_TO_SHOW sont vides le splash-screen au démarrage est désactivé.

_SITE_URL_TO_SHOW

Définir l'adresse de site internet/intranet (URL) à afficher en splash screen ainsi que dans l'onglet à propos (sauf si l'onglet est volontairement masqué en version personnalisée).
Indiquer l'adresse du site web, et non celle du serveur IntraMessenger.
Si cette option ainsi _SITE_TITLE_TO_SHOW sont vides le splash-screen au démarrage est désactivé.

_SKIN_FORCED_COLOR_CUSTOM_VERSION - nouveauté de la version 2.0.4

Option non affichée dans l'ACP.
Trois composantes (de 1 à 255) de la couleur du skin imposé depuis le serveur en version personnalisée.
Nécessite l'activation de l'option _ALLOW_SKIN.
Valeur par défaut : 0-0-0.

_SLOW_NOTIFY - option anciennement nommée _FULL_CHECK jusqu'à la version 2.0.4

Forcer (ou non) à vérifier tout le temps si des contacts sont en attente.
Si activée la vérification n'a lieu que toutes les 3 minutes, permettant ainsi de limiter les accès/requêtes au serveur (surtout si le nombre d'utilisateurs est important).
Voir aussi l'option _CHECK_NEW_MSG_EVERY.

_SPECIAL_MODE_GROUP_COMMUNITY

Activer (ou non) le mode spécial : tout le monde peut voir/discuter avec uniquement les personnes de son (ses mêmes) groupe(s).
La gestion des groupes est centralisée par l'administrateur : voir aussi l'option _GROUP_USER_CAN_JOIN.
Nota : un utilisateur dans aucun groupe ne voit (et n'est vu de) personne.
Voir aussi l'option _GROUP_ID_DEFAULT_FOR_NEW_USER.
Les options : _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY - _GROUP_FOR_SBX_AND_ADMIN_MSG ne doivent pas être activées.
Valeur conseillée : désactivée pour une utilisation classique.

_SPECIAL_MODE_OPEN_COMMUNITY

Activer (ou non) le mode spécial : tout le monde voit/discute avec tout le monde, sans s'ajouter à la liste des contacts.
Les options _SPECIAL_MODE_GROUP_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY ne doivent pas être activée.
Valeur conseillée : désactivée pour une utilisation classique.

_SPECIAL_MODE_OPEN_GROUP_COMMUNITY

Activer (ou non) le mode spécial : tout le monde voit/discute avec tout le monde (sans avoir à s'ajouter à la liste des contacts).
Tri par groupes, gérés par l'administrateur : voir aussi l'option _GROUP_USER_CAN_JOIN.
Voir aussi l'option _GROUP_ID_DEFAULT_FOR_NEW_USER.
Les options : _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_GROUP_COMMUNITY - _GROUP_FOR_SBX_AND_ADMIN_MSG ne doivent pas être activées.
Valeur conseillée : désactivée pour une utilisation classique.

_STATISTICS

Option non affichée dans l'ACP.
Enregistrer (ou non) les statistiques. Consultation depuis l'interface d'admin (ACP).
Valeur conseillée : activée.

_STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL

Option non affichée dans l'ACP.
Définir la nouvelle adresse (url) du nouveau serveur IntraMessenger afin que les utilisateurs y basculent immédiatement au prochain lancement.
Attention, ce n'est pas l'adresse du serveur qui est renommée, c'est bien une nouvelle adresse (en plus) !
Exemple de changement d'adresse sur un même serveur web (donc IntraMessenger installé deux fois) :
    http://votre-serveur/intramessenger/
    http://votre-serveur/im/
Si activée, masque l'option _STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL_AFTER_LOGIN !

_STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL_AFTER_LOGIN - nouveauté de la version 2.0.5

Option non affichée dans l'ACP.
Similaire à l'option _STOP_USE_THIS_SERVER_ADDRESS_NOW_USE_THIS_URL qui ne doit pas être activée.
Définir l'adresse du nouveau serveur afin que les utilisateurs y basculent au prochain lancement : après authentification.
Le changement d'adresse n'est donc connu que des utilisateurs authentifiés.

_TIME_ZONES

Afficher (ou non) les fuseaux/décalages horaires.
Option ajoutée afin de pouvoir désactiver cette fonctionnalité (existant depuis la version 1.3.5).
Valeur conseillée : activée uniquement pour un usage sur internet.

_UNREAD_MESSAGE_VALIDITY

Définir la durée (en jours) de validité des messages non lus (0 : illimité) avant expiration (suppression).
Valeur conseillée : 30.

_USER_HIEARCHIC_MANAGEMENT_BY_ADMIN

Activer (ou non) le mode spécial : gestion hiérarchique des utilisateurs, et afficher la colonne niveau dans l'interface d'administration (ACP).
Les utilisateurs ne peuvent alors plus demander l'ajout d'un contact d'un niveau supérieur.
Les niveaux sont définis dans le fichier de langue du serveur (/common/lang/french-iso-8859-1.inc.php) à la section level.
Les niveaux sont par défaut : Administrateur - Directeur - Chef de service - Employé - Invité (leur nombre est défini avec $c_nb_level).
Un invité ne peut donc pas demandé à s'ajouter mutuellement aux contacts d'un employé, alors que le contraire est possible.
Nécessite l'activation de l'option _ALLOW_MANAGE_CONTACT_LIST.
Les options : _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_OPEN_COMMUNITY - _SPECIAL_MODE_OPEN_GROUP_COMMUNITY ne doivent pas être activées.
Valeur conseillée : désactivée pour une utilisation classique.

_USER_NEED_PASSWORD

Forcer (ou non) l'utilisation d'un mot de passe pour chaque utilisateur.
Peut être désactivée si l'une des options _FORCE_USERNAME_TO_PC_SESSION_NAME ou _PENDING_USER_ON_COMPUTER_CHANGE est activée.
Voir aussi les options : _MINIMUM_PASSWORD_LENGTH - _MAX_PASSWORD_ERRORS_BEFORE_LOCK_USER - _EXTERNAL_AUTHENTICATION - _PASSWORD_FOR_PRIVATE_SERVER.
Valeur conseillée : activée.

_WAIT_STARTUP_IF_SERVER_UNAVAILABLE

Activer (ou non) de laisser au démarrage le client en attente (sans rien signaler) au lieu de se fermer si le serveur est indisponible.
